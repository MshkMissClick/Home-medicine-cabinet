## Введение

Документ описывает план тестирования веб-приложения **MedCabinet**. Цель тестирования – проверить соответствие фактического поведения реализованных функций требованиям и ожиданиям, выявить дефекты до развёртывания.

Основание: анализ исходного кода фронтенда (React: компоненты `AuthPage.tsx`, `Login.tsx`, `Register.tsx`, `HomePage.tsx`, `MedicineForm.tsx`, `MedicineList.tsx`, `MedicineItem.tsx`, `NotificationPopover.tsx`, `NotificationItem.tsx`; Redux слайсы `authSlice.ts`, `medicineSlice.ts`, `notificationSlice.ts`) и бэкенда (Spring Boot: модели `ApplicationArea.java`, `Medicine.java`, `MedicineType.java`, `Notification.java`, `User.java`; контроллеры `NotificationController.java`; репозитории `MedicineRepository.java`).

Ключевой функционал в зоне тестирования: авторизация, управление лекарствами (CRUD, фильтрация), уведомления о сроках годности, ежедневные проверки.

<a name="items"/>

## Объект тестирования

**Фронтенд:** SPA на React с Redux (компоненты для авторизации, списка лекарств, формы, уведомлений).  
**Бэкенд:** Spring Boot REST API с JPA/Hibernate.

Основные REST-эндпоинты (сводно; подробности в приложении `backend_endpoints.csv`):

- `/api/medicines` — CRUD, фильтрация по имени/области/типу, поиск по истекающим срокам.
- `/api/medicine-types` — CRUD.
- `/api/application-areas` — CRUD.
- `/api/notifications` — CRUD, создание уведомлений.
- `/api/auth/login`, `/api/users/register` — авторизация/регистрация.

<a name="quality"/>

## Атрибуты качества

В соответствии с ISO 25010:
- **Функциональная полнота:** Покрытие CRUD для лекарств и уведомлений.
- **Производительность:** Время отклика ≤ 500 мс для API-запросов; обработка 1000 записей без задержек.
- **Совместимость:** Кросс-браузерность (Chrome, Firefox, Edge); API совместимо с JSON.
- **Используемость:** Интуитивный UI, валидация форм.
- **Надёжность:** Обработка ошибок, восстановление после сбоев.
- **Безопасность:** JWT-авторизация, защита от SQL-инъекций.

<a name="risk"/>

## Риски

- **Технические:** Ошибки валидации сроков годности; несоответствие типов в API/фронтенде.
- **Функциональные:** Неправильная фильтрация лекарств или уведомлений; проблемы с датами.
- **Производственные:** Задержки в асинхронных задачах (ежедневные проверки).
- **Безопасность:** Неавторизованный доступ к уведомлениям или лекарствам.
- **Внешние:** Сетевые ошибки, CORS; нестабильная БД.

Митigation: Тестирование на различных сценариях; использование моков.

<a name="features"/>

## Аспекты тестирования

**Функциональные:**
- Авторизация: Регистрация/логин с валидацией, обработка ошибок.
- Лекарства: CRUD, фильтрация по имени/области/типу; поиск по истекающим срокам.
- Уведомления: Создание/удаление, отображение с датами.
- Ежедневные проверки: Генерация уведомлений о сроках годности.

**Нефункциональные:**
- Производительность: Отклик API, обработка больших списков.
- Безопасность: Доступ только авторизованным пользователям.
- Используемость: UI-формы, поиск, диалоги.

<a name="nonfunctional"/>

## Нефункциональные требования

- **Производительность:** Время отклика ≤ 500 мс; обработка 1000 лекарств.
- **Масштабируемость:** Поддержка 100 пользователей.
- **Совместимость:** Браузеры Chrome/Firefox (версии ≥ 100); API JSON.
- **Безопасность:** HTTPS; JWT; валидация входных данных.

<a name="approach"/>

## Подходы к тестированию

- **Юнит-тесты:** Для моделей (валидация), сервисов (бизнес-логика), слайсов Redux.
- **Интеграционные тесты:** API-эндпоинты (CRUD, фильтры) с Postman/Supertest.
- **E2E-тесты:** Cypress для UI (авторизация, добавление лекарства).
- **Ручное тестирование:** UI-интерфейс, поиск, уведомления.
- **Автоматизация:** JUnit для бэкенда, Jest для фронтенда.

<a name="env"/>

## Окружение и данные

- Бэкенд: Java 17, Spring Boot; профиль dev/test (H2/in-memory).
- БД: H2 для тестов, PostgreSQL для prod.
- Фронтенд: Node.js LTS, сборка dev/prod.
- Набор данных: 10 пользователей, 50–200 лекарств, 20 типов/областей, 30 уведомлений.

<a name="acceptance"/>

## Критерии приёмки

- Все CRUD-сценарии для лекарств/уведомлений проходят успешно.
- Фильтрация и уведомления работают без ошибок.
- Нет «критических» дефектов; покрытие ≥ 80%.
- UI адаптивен, без багов.
